# 两步思考（必须执行，不要跳过）

Step 1 提炼内核：用一句话写出原稿想表达的核心观点，存入 `_meta.thinking`。
Step 2 等量或扩写式改写：基于内核做表达、句式、视角的彻底重构，**绝不允许总字数缩水**。

# 硬约束（违反任意一条 → 自检阶段必须重写）

- **总字数**：改写稿字数 ≥ 原稿字数 × 0.95，鼓励落在 [1.00, 1.20] 区间
- **单行字数**：每行字数落在 原行字数 × [0.95, 1.30] 区间
- **行数**：与原稿严格一致，不得合并或拆分
- **ASR 听写错误必须校正**：原稿由自动语音转录生成，常见英文术语 / 品牌名 / 产品名 / 人名的拼写错误（例如 "majourney" → "Midjourney"、"banada" → "Nano Banana"、"二 d" → "2D"、"magic v 七" → "Niji V7"）必须按上下文校正为**正确的官方拼写**；字数比例校验仍以**原稿字数为基准**，校正后字符差异不算违规
- **口语词处理方式**：不是"删除"，而是"改写为书面化表达"
  - 反例：「就是说这其实挺重要的」→「这点重要」（字数缩水，禁止）
  - 正例：「就是说这其实挺重要的」→「这一点的分量，不容小觑」

# 风格规范（可执行的具体要求）

- 每行至少包含 2 个分句，避免一句话直接结尾的短促节奏
- 优先调用：类比、反转、数据点、对比、行业事实
- 严禁：低俗营销词、空洞抒情、连用感叹号；"封神/炸裂/绝杀"等夸张词不要用
- 用书面化句式替代口语词：啊 / 呢 / 吧 / 其实 / 就是说 / 那么 / 这个 / 那个
